# v0.3 重构记录

**日期：** 2026-07-15  
**目标：** 将浏览器 Agent 从单文件、输出驱动的实现重构为可测试、可替换、可演进的分层架构。

## 重构前的问题

- `agent.py` 同时负责环境加载、模型调用、浏览器生命周期、DOM 解析、风险控制、日志和 CLI，职责混杂。
- 文件保留了不可达的旧执行循环及按元素索引操作的旧契约，维护者可能误用。
- TUI 通过重定向进程级 `stdout` 解析运行时输出，界面与业务输出格式紧耦合。

## 新的模块边界

| 模块 | 职责 | 不负责 |
| --- | --- | --- |
| `leafcode.models` | 任务、快照、元素与工具结果的数据契约 | 浏览器、模型、UI |
| `leafcode.browser` | Playwright 会话、稳定元素 ID、动作验证、标签页切换 | 模型决策、安全策略、UI |
| `leafcode.safety` | 高风险动作识别与确认判定 | 执行浏览器操作 |
| `leafcode.runtime` | 计划、循环、状态迁移、调用策略与日志 | Textual 控件渲染 |
| `agent.py` | CLI 兼容入口 | 业务实现 |
| `agent_tui_v4.py` | Textual 输入和显示适配 | 浏览器与模型业务逻辑 |

依赖方向为：入口/UI → runtime → browser/safety/models；`event_log.py` 作为独立基础设施被 runtime 使用。

## 已完成的迁移

- 删除旧的单文件浏览器循环，保留 `agent.py` 的兼容导入与 CLI。
- 将浏览器会话、快照、工具结果与新标签页检测迁移到 `leafcode.browser`。
- 将状态机和风险策略迁移到 `models` / `safety`。
- TUI 改由 `BrowserAgent.on_event` 接收结构化运行时事件，不再重定向全局 `stdout`。
- TUI 将 Playwright 操作固定在单一工作线程；确认界面只提交 Yes/No 信号，由浏览器所属线程继续执行。
- 动态运行文本统一按纯文本渲染，避免动作 JSON 被 Textual 当作富文本标记解析。
- `inform` 任务在完成、失败或取消后自动释放浏览器；`operate` 继续保留会话供后续任务复用。
- 通过模块导入、语法检查和 16 条本地 Playwright/运行时回归验证。

## 2026-07-16 后续修订
