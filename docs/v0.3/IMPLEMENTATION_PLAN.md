# LeafCode v0.3 计划实施文档

## 1. 版本目标

v0.3 的目标是交付一个**可连续使用、执行过程可解释且安全可控**的浏览器 Agent。重点是让 Agent 在同一浏览器会话中稳定完成检索和网页操作，而不是扩充更多站点或堆叠更多 UI 功能。

对应设计说明见 [AGENT_DEVELOPMENT_AND_PLAN.md](AGENT_DEVELOPMENT_AND_PLAN.md)，模块边界见 [ARCHITECTURE.md](ARCHITECTURE.md)。

## 2. 范围与非范围

### 本版本范围

- `operate` 模式下单一浏览器会话中的连续任务；`inform` 在任务结束后释放浏览器。
- 结构化页面快照和稳定元素 ID。
- 动作后验证、失败反馈、重试与取消。
- 风险动作的用户确认。
- OpenCode 风格的任务时间线与终端 UI 改进。
- 可重复的基础任务测试集。

### 本版本不做

- 多 Agent、插件系统、云端同步。
- 自动绕过验证码或网站风控。
- 支付、发布、删除等不可逆操作的自动执行。
- 通用全站登录自动化。

## 3. 实施顺序

功能优先于 UI。只有运行时能稳定地观察、执行和验证，界面美化才不会掩盖失败问题。

```text
M0 基线与测试
  → M1 浏览器执行底座
  → M2 任务状态与安全
  → M3 TUI 信息架构与美化
  → M4 回归验收与发布准备
```

### M1：浏览器执行底座

**目的**：消除“展示的元素”和“实际点击的元素”不一致的问题，并允许在 `operate` 模式跨任务继续操作同一页面。

任务：

- [x] 新建 `BrowserSession`，持有 Playwright、browser、context、page 与当前标签页。
- [x] 将启动、复用、关闭浏览器的生命周期从 `BrowserAgent.run()` 中抽离。
- [x] 定义 `PageSnapshot`：URL、标题、正文摘要、可交互元素、生成时间和可选截图路径。
- [x] 定义 `InteractiveElement`：`id`、角色、文本、属性摘要和可操作方式。
- [x] `observe()` 每次生成新快照；`click/type` 只接受当前快照中的 `element_id`。
- [x] 为 `navigate/click/type/press/scroll` 统一返回 `ToolResult`，包括成功状态、错误、执行前后 URL/标题。
- [x] 增加页面变化检测、工具超时、重复动作限制和新标签页检测。

验收：`operate` 模式下连续两条任务能在同一页面会话完成；点击操作可追溯到快照中的同一个元素 ID；无变化或定位失败时不会被误报为成功。

### M2：状态机、安全与恢复

**目的**：让 Agent 的每个状态可见、可停止、可恢复，且不会未经授权执行高风险动作。

任务：

- [x] 引入 `TaskState`：`planning`、`running`、`awaiting_confirmation`、`completed`、`failed`、`cancelled`。
- [x] 定义 `TaskContext`：原始目标、计划、当前步骤、最近快照、工具事件、来源链接和错误。
- [x] 为 `submit`、下载、上传、发送消息、登录填充等动作标注风险级别。
- [x] 实现 `SafetyPolicy`：低风险自动执行，中高风险生成确认请求。
- [x] TUI 接入确认/拒绝操作；拒绝后可让 Agent 改计划或结束任务。
- [x] 实现 `retry` 与 `continue`：在 `operate` 模式保留当前浏览器页面和已执行步骤；`inform` 任务结束后释放会话。
- [x] 总结结果附来源链接，并明确区分“页面事实”和“Agent 推断”。

验收：取消、失败和确认状态均可在 UI 中正确展示；未确认的风险操作绝不执行；失败后可从当前页面继续。

## 5. v0.3 完成定义

- [ ] 使用 `pip install -r requirements.txt` 和文档步骤可启动项目。
- [x] `operate` 模式下 Agent 可复用同一浏览器会话连续执行至少两条任务；`inform` 在任务结束后关闭浏览器。

- [x] 高风险操作必须停在用户确认状态。

## 7. 风险与决策点

| 风险 | 处理方式 |
| --- | --- |
| 动态页面使元素失效 | 点击前检查快照版本；失效后重新观察再决策 |
| 网站加载慢或弹窗干扰 | 统一等待策略、超时和弹窗/新标签页事件处理 |
| LLM 输出不合法 | 使用 JSON schema 或工具调用校验；失败时只重试有限次数 |
| 页面提示注入攻击 | 页面文本作为不可信观察数据；系统规则和安全策略不可被网页覆盖 |
| UI 与运行时耦合 | 运行时仅发布事件，Textual 订阅事件渲染，不直接承载业务逻辑 |
| 范围失控 | M1/M2 未验收前，不加入多 Agent、插件和复杂登录能力 |
