# Browser Agent 架构重构计划 v4

## 当前问题诊断

1. **TextArea 不工作**: `TextArea.code_editor()` 在 prompt_toolkit 布局中有兼容问题，输入无法正常提交和显示
2. **Tab 冲突**: Tab 同时绑定切换焦点和切换模式，两套逻辑冲突
3. **布局问题**: 输入区虽然 CSS 定义了，但实际渲染时不可交互
4. **文档散落**: 规划文件与代码混在同一目录

## 架构重构方案

### 1. 布局重新设计

```
┌─────────────────────────────────────────────┐
│  ⏳ Preparing... (加载条，loading 时显示)      │
├─────────────────────────────────────────────┤
│                                             │
│  消息区（可滚动）- 显示用户消息和AI回复         │
│                                             │
├─────────────────────────────────────────────┤
│  ┌──────────────────────────────────────┐   │
│  │  输入区 (TextArea, 高度 auto)         │   │
│  └──────────────────────────────────────┘   │
│  INFORM  OPERATE   [Ctrl+Enter 提交]         │
├─────────────────────────────────────────────┤
│  (底部留白 1 行)                             │
└─────────────────────────────────────────────┘
```

### 2. 键位映射重设计

| 键位 | 动作 | 说明 |
|------|------|------|
| Ctrl+Enter | 提交输入 | 从 TextArea 提取文本并执行 |
| Ctrl+Tab | 切换模式 | inform ↔ operate |
| Tab | 切换焦点 | 消息区 ↔ 输入区 |
| Esc (输入区) | Esc 计数 | 3次连按中断任务 |
| Esc (消息区) | 回输入区 | 焦点回到输入框 |

### 3. 文本编辑器选型

放弃 TextArea.code_editor，改用纯 TextArea 或返回 Input+更灵活的布局：

**方案 A**: TextArea（多行） + Ctrl+Enter 提交
**方案 B**: Input（单行，回退） + Enter 提交

推荐方案 B（先回退到可靠方案），TextArea 多行在 prompt_toolkit 中存在已知问题。

### 4. 模式切换 UI

- 底部两个标签：`INFORM`（默认高亮）和 `OPERATE`
- Ctrl+Tab 键位切换，同时更新标签高亮
- 标签本身可点击切换

### 5. Agent 双模式行为

| 维度 | inform | operate |
|------|--------|---------|
| 目标 | 搜索并提取信息 | 导航到目标页面状态 |
| 终止 | done 后关闭浏览器 | done 后保留浏览器 |
| 结果 | 文本总结 | 浏览器窗口留给用户 |

## 实施计划

### 第一步：回退到可靠基础
- 用 `Input` 替代 `TextArea`（确保交互正常）
- 布局简化：消息区 + 输入区 + 模式标签 + 底部留白

### 第二步：模式切换
- Ctrl+Tab 切换
- 标签点击切换
- 模式状态传递给 Agent

### 第三步：operate 模式行为
- Agent 支持 mode 参数
- 凭证读取
- done 后不关闭浏览器

### 第四步：后续扩展
- TextArea 多行支持（等 prompt_toolkit 稳定方案）
- 登录自动化
- 更多网站凭证支持
